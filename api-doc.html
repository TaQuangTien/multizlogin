1. findUser

Mục đích: Tìm kiếm người dùng dựa trên số điện thoại.

Endpoint: POST /findUser

Parameters:

FieldLoạiMô tảBắt buộc
phone string Số điện thoại của người dùng. Có
ownId string ID tài khoản Zalo đang đăng nhập (lấy từ /accounts). Có

Ví dụ:

{
  "phone": "0123456789",
  "ownId": "0000000000000000001"
}

Ví dụ curl:

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "phone": "0123456789",
  "ownId": "0000000000000000001"
}' http://localhost:3000/findUser

4. sendMessage

9. sendImageToUser

Mục đích: Gửi một hình ảnh đến một người dùng Zalo.

Endpoint: POST /sendImageToUser

Header: Yêu cầu x-api-key để xác thực.

FieldLoạiMô tảBắt buộc
imagePath string Đường dẫn hoặc URL của hình ảnh (hỗ trợ JPEG, PNG, GIF). Không (yêu cầu một trong imagePath hoặc imageData)
imageData string Dữ liệu hình ảnh dạng base64, phải có prefix MIME type (ví dụ: data:image/jpeg;base64,...). Hỗ trợ JPEG, PNG, GIF. Không (yêu cầu một trong imagePath hoặc imageData)
threadId string ID của người dùng nhận tin nhắn. Có
ownId string ID tài khoản Zalo gửi tin nhắn (lấy từ /accounts). Có

Ghi chú: Hình ảnh từ base64 sẽ được lưu tạm với định dạng tương ứng (JPEG, PNG, GIF) trước khi gửi. Đảm bảo cung cấp prefix MIME type trong imageData để xác định đúng định dạng.

Ví dụ (sử dụng imagePath):

{
  "imagePath": "https://example.com/image.jpg",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ (sử dụng imageData):

{
  "imageData": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD...",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ curl (sử dụng imagePath):

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "imagePath": "https://example.com/image.jpg",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendImageToUser

Ví dụ curl (sử dụng imageData):

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "imageData": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD...",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendImageToUser

13. sendFileToUser

Mục đích: Gửi một file đến một người dùng Zalo.

Endpoint: POST /sendFileToUser

Header: Yêu cầu x-api-key để xác thực.

FieldLoạiMô tảBắt buộc
filePath string Đường dẫn hoặc URL của file (hỗ trợ PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX, TXT, CSV, ZIP, RAR, 7Z, MP3, WAV, MP4, AVI, MOV và các định dạng hình ảnh). Không (yêu cầu một trong filePath hoặc fileData)
fileData string Dữ liệu file dạng base64. Có thể có prefix MIME type (ví dụ: data:application/pdf;base64,...) hoặc chỉ là base64 thuần. Không (yêu cầu một trong filePath hoặc fileData)
mimeType string MIME type của file (ví dụ: application/pdf, application/vnd.openxmlformats-officedocument.wordprocessingml.document). Chỉ cần thiết khi fileData không có prefix MIME type. Không
threadId string ID của người dùng nhận file. Có
ownId string ID tài khoản Zalo gửi file (lấy từ /accounts). Có

Các MIME type được hỗ trợ:

  • Documents: application/pdf, application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document, application/vnd.ms-excel, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet, application/vnd.ms-powerpoint, application/vnd.openxmlformats-officedocument.presentationml.presentation, text/plain, text/csv
  • Archives: application/zip, application/x-rar-compressed, application/x-7z-compressed
  • Audio: audio/mpeg, audio/wav, audio/ogg, audio/mp4
  • Video: video/mp4, video/avi, video/quicktime, video/x-msvideo
  • Images: image/jpeg, image/png, image/gif, image/webp, image/svg+xml

Ghi chú: File từ base64 sẽ được lưu tạm với định dạng tương ứng trước khi gửi. Nếu không xác định được MIME type, file sẽ được lưu với extension .bin.

Ví dụ (sử dụng filePath):

{
  "filePath": "https://example.com/document.pdf",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ (sử dụng fileData với prefix):

{
  "fileData": "data:application/pdf;base64,JVBERi0xLjQKJeLjz9MKMSAwIG9iago8PAov...",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ (sử dụng fileData không có prefix + mimeType):

{
  "fileData": "JVBERi0xLjQKJeLjz9MKMSAwIG9iago8PAov...",
  "mimeType": "application/pdf",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ curl (sử dụng filePath):

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "filePath": "https://example.com/document.pdf",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendFileToUser

Ví dụ curl (sử dụng fileData với prefix):

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "fileData": "data:application/pdf;base64,JVBERi0xLjQKJeLjz9MKMSAwIG9iago8PAov...",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendFileToUser

Ví dụ curl (sử dụng fileData + mimeType):

curl -X POST -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "fileData": "JVBERi0xLjQKJeLjz9MKMSAwIG9iago8PAov...",
  "mimeType": "application/pdf",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendFileToUser

14. accounts

Endpoint: GET /accounts

Header: Yêu cầu x-api-key để xác thực.

Response:

[
  {
    "ownId": "0000000000000000001",
    "proxy": "http://proxy.example.com",
    "phoneNumber": "0123456789"
  },
  {
    "ownId": "0000000000000000002",
    "proxy": null,
    "phoneNumber": "N/A"
  }
]

Ví dụ curl:

curl -H "x-api-key: YOUR_API_KEY" http://localhost:3000/accounts
