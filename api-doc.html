Phiên bản: v{{APP_VERSION}} Phát triển bởi: Tiến Tạ Năm: 2026 Powered by: zca-js (unofficial Zalo API)
Lưu ý: Dự án sử dụng API không chính thức của Zalo. Việc sử dụng có thể vi phạm điều khoản dịch vụ của Zalo và dẫn đến việc tài khoản bị khóa. Hãy cân nhắc kỹ trước khi triển khai.
Xác thực API: Các endpoint yêu cầu 2 lớp xác thực:
  • Header x-api-key: YOUR_API_KEY (Cấu hình qua X_API_KEY).
  • Basic Auth: Header Authorization chứa username/password (Cấu hình qua ADMIN_USERNAME và ADMIN_PASSWORD).
Cấu hình Self Listen: Sử dụng biến môi trường SELF_LISTEN=true nếu bạn muốn bot nhận được cả các tin nhắn do chính nó gửi đi. Mặc định là false.

1. findUser POST

Mục đích: Tìm kiếm người dùng dựa trên số điện thoại.

Endpoint: POST /findUser

Parameters:

Field Loại Mô tả Bắt buộc
phone string Số điện thoại của người dùng. Có
ownId string ID tài khoản Zalo đang đăng nhập (lấy từ /accounts). Có

Ví dụ:

{
  "phone": "0123456789",
  "ownId": "0000000000000000001"
}

Ví dụ curl:

curl -X POST -u ADMIN_USER:ADMIN_PASS -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "phone": "0123456789",
  "ownId": "0000000000000000001"
}' http://localhost:3000/findUser

Endpoint: POST /sendmessage

Field Loại Mô tả Bắt buộc

  "type": "0",
  "ownId": "0000000000000000009",
  "imagePath": "https://example.com/image.jpg"
}

10. sendImageToUser POST

Mục đích: Gửi một hình ảnh đến một người dùng Zalo.

Endpoint: POST /sendImageToUser

Header: Yêu cầu x-api-key để xác thực.

Field Loại Mô tả Bắt buộc
imagePath string Đường dẫn hoặc URL của hình ảnh (hỗ trợ JPEG, PNG, GIF). Không (yêu cầu một trong imagePath hoặc imageData)
imageData string Dữ liệu hình ảnh dạng base64, phải có prefix MIME type (ví dụ: data:image/jpeg;base64,...). Hỗ trợ JPEG, PNG, GIF. Không (yêu cầu một trong imagePath hoặc imageData)
threadId string ID của người dùng nhận tin nhắn. Có
ownId string ID tài khoản Zalo gửi tin nhắn (lấy từ /accounts). Có

Ghi chú: Hình ảnh từ base64 sẽ được lưu tạm với định dạng tương ứng (JPEG, PNG, GIF) trước khi gửi. Đảm bảo cung cấp prefix MIME type trong imageData để xác định đúng định dạng.

Ví dụ (sử dụng imagePath):

{
  "imagePath": "https://example.com/image.jpg",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ (sử dụng imageData):

{
  "imageData": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD...",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}

Ví dụ curl (sử dụng imagePath):

curl -X POST -u ADMIN_USER:ADMIN_PASS -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{

12. sendFileToUser POST

Mục đích: Gửi file tài liệu, nén, audio hoặc video đến người dùng.

Endpoint: POST /sendFileToUser

Field Loại Mô tả Bắt buộc
filePath string Đường dẫn hoặc URL của file. Không (yêu cầu một trong filePath hoặc fileData)
fileData string Dữ liệu file dạng base64. Không (yêu cầu một trong filePath hoặc fileData)
mimeType string MIME type của file (ví dụ: application/pdf). Không
threadId string ID của người dùng nhận tin nhắn. Có
ownId string ID tài khoản Zalo gửi tin (lấy từ /accounts). Có
Định dạng hỗ trợ:
  • Tài liệu: .pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx
  • Văn bản: .txt, .csv, .json, .xml
  • Nén: .zip, .rar, .7z
  • Âm thanh: .mp3, .wav, .ogg, .m4a
  • Video: .mp4, .avi, .mov
  • Ảnh: .jpg, .png, .gif, .webp, .svg
* Các định dạng khác sẽ được gửi dưới dạng file binary (.bin)

Ví dụ curl (sử dụng filePath/URL):

curl -X POST -u ADMIN_USER:ADMIN_PASS -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "filePath": "https://example.com/document.pdf",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendFileToUser

Ví dụ curl (sử dụng fileData Base64):

curl -X POST -u ADMIN_USER:ADMIN_PASS -H "Content-Type: application/json" -H "x-api-key: YOUR_API_KEY" -d '{
  "fileData": "SGVsbG8gdHUgVMO0bSB0ZXN0Cg==",
  "mimeType": "text/plain",
  "threadId": "0000000000000000001",
  "ownId": "0000000000000000009"
}' http://localhost:3000/sendFileToUser
